Enhance the log out tracking

Status: In this build

The fix we put in 2 weeks ago does not work with web-kit browsers. Chrome and Safari don't allow asynchronous events to complete in the onWindowClosing method thus the system never gets the log out message. We have a fix using a synchronous call implemented with jQuery. This will require the use of GWT native methods.

Performanace improvements:

Status: 1. in this build. Used on the search screen. Compare this with the bookcase display.

  • 1. Pre-built images of books on the server. That way we can control to size of the download for the display and the front end will not need to create the image on the fly. This may result in slightly less detailed images but provide a more consistent display with lees waiting for the user
  • 2. Scrollable search results return. Only bring back as many results as the user can see in the view. As the user scrolls in the view bring back additional results. This will provide a faster initial display.
  • 3. Load the bookcase panels (My Books, Books I'm Co-Authoring, Books I Like) separately when the user navigates to the panel. Again, this will improve the initial response time.

Implement Co-Author workflow

This will require a way for authors to lookup potential co-authors and send them a message. We'll need a way for the co-author to respond to the request and the author to approve of deny the response from the co-author. At this time we will be implementing the user story where the author initiates the request. At some later date we can implement the user story where the co-author initiates the request to the author.
